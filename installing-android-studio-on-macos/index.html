Installing Android Studio on macOS

note

These instructions were written for Buzztouch 5.0 for Android.

Installing Android Studio​

Download Android Studio https://developer.android.com/studio

(Choose Intel Mac or Apple Silicon Mac depending on which you have)

Open .dmg you downloaded and drag Android Studio to the Applications folder.

Navigate to your Applications folder and open Android Studio. Gatekeeper may prompt you to confirm you want to open it.

When opening for the first time, Android Studio will ask you to if you’d like to import any existing installation or config.

In my case I didn’t have an existing configuration… so I left Do not import settings selected and clicked OK to proceed.

I opted out of sharing analytics by choosing the Don’t send option.

Android Studio will open an Auto Setup Wizard. We’ll walk through the following steps

Click Next

Standard is auto-selected, click Next

Choose your preferred theme, then click Next Once complete, Android Studios will prompt to verify your Settings. Click Next to proceed

You will be prompted to accept the License Agreement. Select the Accept radio button, then press Finish to begin installation.

Components will begin downloading…

Once complete, the Android Studio Welcome pane will open.

Downloading Your Android Studio Project​

Visit the Download Project page for your Buzztouch application.

Select the [Android Beta] Buzztouch Core v5.0.0 build version, then click Package Android Studio Project.

Once the packaging is complete, click Download .zip archive to download your Android Studio project.

Opening Your Buzztouch Project​

Unzip your Android Studio project, then drag your Buzztouch project folder on to the Android Studio app icon.

You may be prompted to allow Android Studio to access your downloads folder.

If prompted, click Trust Project to allow your project to open in Android Studio.

Your project will begin loading… you will see several Grade configurations downloading in the bottom right hand corner.

Installing Android Emulators​

Under the Tools menu select Device Manager

In the sidebar pane that opens, click the Create device button.

The Virtual Device Configuration window will open.

In this case, In this example, we chose the Pixel 6 Pro. Click Next to begin installation.

Android Studio will request you choose an System Image (operating system) to install on the Emulator. UpsideDownCake is currently the latest release, (you may want to pick a previous version such as Tiramisu if you aren’t comfortable modifying your gradle config). If the image is not installed you will not be able to click the Next button, instead click the download icon next to the Release Name. This will bring up the SDK Component Installer.

You must agree to the license agreement to install the operating system. Choose Accept then click Next to continue.

Once the install is complete, click Finish to close the dialog

You can now use the System Image you installed on the emulator. Click Next to proceed.

If you’d like you can name your AVD (Android Virtual Device) or you can leave it with the default name. Click Finish to create the emulator.

Once the emulator is installed, you can click the ▶ icon to build and run your project on the Android Virtual Device.
